# Портфолио — монохром / swiss editorial

Одностраничный сайт-портфолио на чистом HTML / CSS / JS. Без сборки, без зависимостей.

## Структура

```
portfolio/
├── index.html      # разметка всех секций
├── styles.css      # стили (дизайн-токены вверху файла)
├── script.js       # reveal-анимации, фильтры, меню, видео
├── assets/         # сюда кладите свои фото и видео
└── README.md
```

## Что заменить на своё

Найдите в `index.html` плейсхолдеры в квадратных скобках:

- `[ИМЯ]` — ваше имя (встречается в `<title>`, навбаре, футере)
- Слоган в Hero: `ДЕЛАЮ / ЦИФРОВЫЕ / ПРОДУКТЫ`
- Текст «Обо мне», цифры в статистике
- Цены в секции «Услуги»
- Контакты: `t.me/USERNAME` и `mail@example.com` (замените везде)

## Картинки и видео

Сейчас в качестве примера подключены фото с Unsplash. Замените их на свои:

1. Положите файлы в папку `assets/` (например `assets/work-1.jpg`, `assets/demo.mp4`).
2. В `index.html` поменяйте `src="..."` на `src="assets/ваш-файл.jpg"`.

Все изображения и видео автоматически становятся **чёрно-белыми** (`filter: grayscale(1)` в CSS) — отдельно обрабатывать не нужно.

### Видео на интро-заставке (приветствие при загрузке)
При входе показывается заставка: видео по центру в рамке со светящимся анимированным свечением, под ним подпись «ещё один сайт для ещё одного моего проекта». После окончания ролика заставка плавно уезжает и открывается сайт.

1. Положите свой ролик в `assets/intro.mp4` (MP4 / H.264, без звука, 3–8 сек).
2. Поменяйте `poster="..."` на свой кадр-превью (необязательно).
3. Заставка показывается **один раз за сессию** (в рамках вкладки). Чтобы увидеть её снова — откройте сайт в новой вкладке или очистите sessionStorage в DevTools.

Подпись меняется в `index.html` → элемент с `id="introCaption"`.
Если файла видео нет — заставка просто закроется через 5 секунд (poster останется виден).

### Видео вместо фото (работы + «Обо мне»)
Все медиа-боксы — это SVG-плейсхолдер (кружок с ▶ и подписью «ВИДЕО»), поверх которого лежит `<video>`. Пока файла нет — виден плейсхолдер; как появится — видео его перекрывает. Имена файлов, которые ждёт разметка:

```
assets/work-1.mp4 ... work-6.mp4   — карточки в секции «Работы»
assets/about.mp4                   — видео в секции «Обо мне» (autoplay, без звука)
```

Видео в карточках играют при наведении, в «Обо мне» — автоматически по кругу. Всё авто-ч/б.

### Видео в карточках (подробнее)
Карточка с видео уже есть в разметке (второй проект). Видео:
- беззвучное, зацикленное;
- **проигрывается при наведении** курсора, на паузе — когда уходите;
- `poster` — превью до загрузки.

```html
<div class="media media--card">
  <video src="assets/demo.mp4" muted loop playsinline poster="assets/demo-poster.jpg"></video>
</div>
```

## Фильтры работ

У каждой карточки есть `data-cat` (`site` / `bot` / `auto`). Кнопки-чипы фильтруют по этому атрибуту. Чтобы добавить категорию — добавьте чип в `.chips` с `data-filter` и карточки с тем же `data-cat`.

## Локальный запуск

Просто откройте `index.html` в браузере. Либо лёгкий сервер:

```bash
# Python
python -m http.server 5173
# затем откройте http://localhost:5173
```

## Деплой

### Vercel
1. Залейте папку в GitHub-репозиторий.
2. На vercel.com → New Project → импортируйте репозиторий.
3. Framework Preset: **Other**, остальное оставьте пустым → Deploy.

Либо через CLI из папки `portfolio`:
```bash
npx vercel
```

### GitHub Pages
1. Запушьте проект в репозиторий.
2. Settings → Pages → Source: **Deploy from a branch** → branch `main`, папка `/root` (если файлы в корне) или `/docs`.
3. Сайт будет доступен по `https://USERNAME.github.io/REPO/`.

> Если кладёте в подпапку (как `portfolio/`), на GitHub Pages удобнее переместить файлы в корень репозитория или настроить папку публикации.

## Доступность и производительность

- Семантическая разметка (`header`, `main`, `section`, `footer`).
- Шрифты грузятся с `display=swap`.
- Анимации отключаются при `prefers-reduced-motion`.
- Нет внешних JS-библиотек — быстрая загрузка.
